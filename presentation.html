Welcoming everyone and short introduction

Overview of the differents aspects we will review in this presentation

  • A chatbot is software that simulates human conversation, via text or voice, so people can interact with digital services naturally.
  • They range from simple, rule-driven responders to advanced systems that learn and personalize.

  • Main goals: efficiency (automation of repetitive queries), engagement (24/7 consistent support), and cost savings.
  • They can also drive lead generation by qualifying users and guiding them through funnels.

  • Early era: ELIZA in the 1960s - rule-based pattern matching; no real understanding.
  • ML era: NLP + machine learning improved intent recognition, still domain limited.
  • Today: Transformers & LLMs enable context understanding, reasoning, and generative replies.

  • Rule-based = decision trees/if else - reliable for predictable flows.
  • Generative (LLM) = flexible, human-like text, but can "hallucinate" if unchecked.
  • RAG = retrieves facts from a knowledge base first, then generates - reducing errors.

  • Task-oriented bots complete specific jobs (bookings, orders, support).
  • Open-domain bots chat broadly and keep a natural flow without a single goal.

  • Text bots live on web, apps, and messaging platforms.
  • Voice assistants (e.g., Siri, Alexa) use ASR for input and TTS for responses - great for hands-free tasks.

  • Three forces: cloud computing for scalable compute, lots of data for training, and LLM breakthroughs for capability jumps.
  • This evolution moved bots from scripted tools to powerful assistants.

  • User sends a message through a channel (e.g., WhatsApp, web).
  • Platform receives it via webhooks/APIs and routes it to the backend for processing.
  • System interprets intent, calls data/tools, then responds via HTTP or WebSocket; voice systems use ASR/TTS in the loop.

  • Cloud-hosted services on AWS, Azure, or GCP; APIs over HTTP connect services together.
  • Core pieces: NLU (intent/entities), dialogue management (state/flow), NLG (natural replies), tool use/agents, and orchestration.
  • Backends include databases, vector stores for semantic search, analytics, and observability tools.

  • Transformers - Modern type of AI model that can understand and create human-like language.
  • Prompting - Giving the AI instructions or examples to guide it to respond the way you want.
  • RAG - Retrieval-Augmented Generation; the AI fetches extra info from external sources before answering, making responses more accurate and up to date.
  • Fine-tuning - The model is trained on specific data (e.g., company data) to perform better in that particular domain.
  • Safety basics - Filters and checks are used to prevent the AI from generating harmful or inappropriate content.

Channels: Places where chatbots live and interact with users.

  • Messaging apps - Slack, WhatsApp, etc.
  • Voice assistants - Google Assistant or Alexa, which use TTS (text-to-speech) and STT (speech-to-text) to communicate.
  • Websites - Chatbots can be embedded directly on a page to assist visitors.

Protocols: The rules and methods chatbots use to send and receive messages.

  • HTTP/Web - The basic way data is sent and received over the internet.
  • Webhooks - Allow chatbots to receive automatic updates or notifications from other systems.
  • WebSockets - Enable real-time communication, useful for live customer support chats.

The backend is the behind-the-scenes part of the chatbot - where data is stored, tools are used, and performance is tracked.

  • Databases - Store user information and chat history.
  • Vector databases - Help the chatbot understand the meaning and context of user language.
  • APIs and tool calling - Allow the chatbot to perform actions such as booking appointments or checking the weather.
  • Observability tools - Monitor chatbot performance and help identify or fix issues when needed.

This shows the chatbot conversation flow step by step:

  1. User sends a message (e.g., on WhatsApp).
  2. The chatbot receives the message via a webhook.
  3. The chatbot checks a database or uses an API to find the needed information.
  4. It then sends a quick reply back to the user via webhook or HTTP.
  5. Voice chatbots use speech-to-text (STT) to understand users and text-to-speech (TTS) to respond out loud.

Running chatbots with large language models can be costly since usage is billed per token. To optimize performance and cost:

  • Caching stores frequent answers to avoid regeneration.
  • Model routing sends simple queries to smaller models and complex ones to larger ones.
  • Monitoring & scaling tools ensure fast, reliable responses and automatic adaptation to traffic.
The goal is balancing speed, cost, and reliability.

Chatbots rely on several tools and platforms to operate effectively:

  • Frameworks (LangChain, Rasa, Botpress) for conversation flow and integrations.
  • Cloud AI platforms (OpenAI, Azure AI, Google Vertex AI) to host models and handle heavy compute.
  • Vector databases (Pinecone, Weaviate) to enable Retrieval-Augmented Generation (RAG) - finding relevant info before answering.
  • LangOps: an emerging practice for managing prompts, analytics, and performance - like DevOps for language models.

Demo - building a simple chatbot with Chatling:

  • Add data such as documents or FAQs so the bot can learn from them.
  • Customize tone and language to match the brand.
  • Integrate the bot into websites or apps (Slack, WhatsApp).
  • Test, monitor user interactions, and deploy the bot live.
A quick no-code way to build a working chatbot, simple but powerful.

Future trends & wrap-up:

  • Multimodal chatbots combining text, voice, and images for more natural communication.
  • Lightweight, on-device bots running on phones or computers for speed and privacy.
  • More personalized and adaptive experiences that remember users over time.
  • Strong focus on ethics and transparency, users should know when they're interacting with AI and how their data is used.
In short, chatbots are evolving from simple scripted tools into smart, scalable, and more human-like assistants.
